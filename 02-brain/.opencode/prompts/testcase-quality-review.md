# Testcase Quality Review Prompt

Use this prompt to review existing testcase quality using ISQA governance and FALID rules.

## Output

- weak steps
- ambiguous expected result
- missing data prerequisite
- missing evidence expectation
- risk priority mismatch
- suggested improvement
- whether update requires confirmation

## Rules

- Do not modify spreadsheet.
- Do not treat ISQA as business truth.
- Suggest improvements only.
- Keep testcase source, UI actual, Figma, MoM, BPMN, and stakeholder confirmation separated.
- If evidence conflicts, route to Needs Confirmation instead of changing the testcase definition.
