# Runtime Workspace

This folder is organized by function so active testing code, session helpers, and operator docs do not pile up in one place.

Start with:

- `docs/ACTIVE_RUNTIME_SURFACE.md`
- `docs/READY_COMMANDS.md`
- `docs/RUNTIME_STRUCTURE.md`

Top-level folders:

- `access/` for app entry and generic access checks
- `capture/` for manual evidence recording
- `docs/` for runtime truth and operating guidance
- `modules/` for active suites and per-module probes
- `session/` for runtime session entrypoints and wrappers
- `shell/` for lightweight PowerShell helpers
